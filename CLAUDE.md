# Koncierge — Kapable Onboarding Assistant

## What This Is

An AI onboarding concierge for the Kapable platform. Embedded in the console UI, it guides new team members through the platform with contextual help, navigation, and voice.

## Project Structure

- `knowledge/` — Pre-computed platform knowledge (the Koncierge's brain)
- `knowledge/subsystems/` — Per-subsystem exploration outputs
- `knowledge/KAPABLE_KNOWLEDGE_BASE.md` — Merged document loaded as system prompt
- `agents/` — Agent definitions (koncierge, explorer)
- `src/` — Agent server code (future)
- `PLAN.md` — Full architecture and phased plan

## Knowledge Distillation

The knowledge base is generated by spawning explorer agents that read each subsystem and produce structured documentation. To regenerate:

1. Run explorer agents (one per subsystem) against the live codebase
2. Each produces a markdown doc in `knowledge/subsystems/`
3. Merge into `knowledge/KAPABLE_KNOWLEDGE_BASE.md`

## Koncierge Integration Checklist

To verify the Koncierge panel is correctly integrated into the console, check these five points:

1. **routes.ts** — `api.koncierge.message` and `api.koncierge.health` routes exist in `dev.kapable.console/app/routes/`
2. **_app.tsx** — `KonciergePanel` is rendered inside the app layout (imported from `@kapable/koncierge/ui`)
3. **package.json** — `@kapable/koncierge` is listed as a `file:` dependency in `dev.kapable.console/package.json`
4. **Env vars** — `KONCIERGE_URL` and `KONCIERGE_SECRET` are set in the production environment (verify via `curl https://console.kapable.dev/api/koncierge/health`)
5. **Smoke test** — Health endpoint returns `{"status":"ok","backend":{"status":"ok"},"integration":{"panel_mounted":true}}`
